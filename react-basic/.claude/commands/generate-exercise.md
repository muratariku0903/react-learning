# 演習作成コマンド

トピック名: $ARGUMENTS

## 実行手順

1. `docs/roadmap/$ARGUMENTS.md` ファイルを読み込む
2. ロードマップの内容を分析し、サブトピックを特定する
3. CLAUDE.mdの「演習作成時の進め方（ワークフロー）」に従って演習を作成する
4. トピックディレクトリ直下に `progress.md`（進捗管理ファイル）を作成する

## ディレクトリ構成

```
src/exercises/
  └── [トピック名]/
      ├── progress.md        ← 進捗管理ファイル
      ├── [サブトピック1]/
      │   ├── README.md
      │   ├── answer.md
      │   ├── design.md
      │   ├── notes.md
      │   └── App.tsx
      ├── [サブトピック2]/
      │   └── ...
      └── [サブトピック3]/
          └── ...
```

## 進捗管理ファイル（progress.md）

トピックディレクトリ直下に作成し、各サブトピックの完了状況をチェックボックスで管理する。

### フォーマット

```markdown
# [トピック名] 進捗管理

## サブトピック一覧

- [ ] [サブトピック1]
- [ ] [サブトピック2]
- [ ] [サブトピック3]
```

### 使い方

- 演習が完了したら `[ ]` を `[x]` に変更する
- 例: `- [x] サブトピック1`

## 各サブトピックディレクトリに作成するファイル

- `README.md` - 課題文（目的・要件・制約・ヒント）
  - 要件には「言語化演習」と「実装演習」の両方を含める
- `answer.md` - 言語化演習の回答フォーマット
- `design.md` - 実装演習の設計メモ用（空ファイル）
- `notes.md` - 学びログ用（空ファイル）
- `App.tsx` - 最小構成の実装

## 注意事項

- `docs/roadmap/$ARGUMENTS.md` が存在しない場合はエラーメッセージを表示
- 画像参照（`attachment:`や`image.png`）は無視する
- ロードマップの章立てや内容からサブトピックを適切に分割する
- 過剰な設計や抽象化は行わない
