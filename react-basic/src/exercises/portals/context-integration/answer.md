# 言語化演習 回答

## Q1. なぜPortalでもContextが使えるのか

Contextが参照可能となるのはReactツリーとしてProviderでラップされてるコンポーネントであるので、たとえ、描画先が別のDOMノードに移ったとしてもContextは参照可能。




## Q2. Portalの「変わること」と「変わらないこと」

### DOMツリー上の位置

- 変わる:
- 理由:描画先をPortalで指定したDOMに変更可能

### Reactコンポーネントツリー上の位置

- 変わらない:
- 理由:Portalで変更するのはDOMの描画先だけであり、Reactから見たコンポーネントの関係は変わらないため

### propsの受け渡し

- 変わらない:
- 理由:Reactツリー上の親子関係が保たれているので通常通り扱える

### stateの共有

- 変わらない:
- 理由:Reactツリー上の親子関係が保たれているので通常通り扱える

### Contextの継承

- 変わらない:
- 理由:Reactツリー上の親子関係が保たれているので通常通り扱える

### CSSの継承（親要素のスタイル）

- 変わる:
- 理由:描画先が変わるので、親要素のスタイルを継承しない
