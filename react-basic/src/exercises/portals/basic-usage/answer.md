# 言語化演習 回答

## Q1. Portalsが解決する問題

通常、Reactではコンポーネントは親コンポーネントの制約を受ける。例えば、z-indexやoverflow-hiddenなどのCSSなどが親に指定されていた場合、それが子コンポーネントにも受け継がれる。しかし、Portalsを使うことで、描画先を親の配下ではなく、別のDOMに移し替えることで、そうした制約を回避することができる。

課題の例では、モーダルが親要素のoverflow:hiddenやheightのスタイルを引き継ぐので、例えば、モーダルを親要素の範囲を超えて表示させたりといったことができなくなってしまう。



## Q2. createPortalの役割

一つ目の引数は描画対象となるReact要素で、二つ目は一つ目で指定したDOMの描画先となるノード。

回答は正しいです。Portalsの本質的な役割（DOM階層の制約からの解放）を理解されています。

補足として、Portalsの重要な特性を一つ追加します：
- **DOMツリー上では親の外に描画されるが、Reactのコンポーネントツリー上では元の位置に留まる**

これは次の演習（event-bubbling）で詳しく扱いますが、イベントのバブリングはReactのコンポーネントツリーに従って伝播するため、Portalで描画された要素のイベントも親コンポーネントでキャッチできます。
