# 回答

## 質問1: props drilling問題の説明

UserMenuコンポーネントにpropsを渡すために、そのpropsを全く使わないコンポーネント（LayoutやHeader）にpropsを渡すことになる。
LayoutやHeaderコンポーネントの可読性が低下し、依存性が高くなり、再利用性が低下する。

---

## 質問2: 「drilling」という名前の由来

「drilling」とは穴を掘るという意味で、propsが親から子に渡っていく様子が「穴を掘る」ことに似ていることが由来。

---

## 質問3: 各観点からの問題点

### コードの可読性について

処理の中で使わないpropsを受け取ると、何も知らない開発者はそのpropsを使うものだと考えるので、コンポーネントの依存関係を正しく把握できなくなる。

### メンテナンス性について

propsを中継してる全てのコンポーネントに修正が入るので、メンテナンス性が低下する。

### コンポーネントの再利用性について

不要なpropsを受け取るせいで、再利用する側でも、同じpropsを渡さないといけなくなり、
結果的に、再利用しづらい状況となってしまう。
