# Props Drilling（プロップスドリリング）問題の理解

## 目的

Reactにおけるprops drilling問題を理解し、なぜContextが必要とされるのかを体感する。

---

## 言語化演習

### 質問1
以下のコンポーネント構造があるとします。

```
App
 └─ Layout
     └─ Header
         └─ UserMenu
```

`App`コンポーネントが持っている`user`オブジェクトを`UserMenu`で表示したい場合、propsだけで解決しようとすると何が起きますか？この問題点を自分の言葉で説明してください。

### 質問2
「props drilling」という名前の由来について考えてみてください。なぜ「drilling（穴を開ける）」という表現が使われているのでしょうか？

### 質問3
props drillingが発生している状態は、以下のどのような観点から問題があると考えられますか？

- コードの可読性
- メンテナンス性
- コンポーネントの再利用性

それぞれについて、具体的にどんな問題が起きうるか考えてください。

---

## 実装演習

### 課題

現在の`App.tsx`には、props drillingが発生している実装があります。

1. まず現状のコードを読んで、どのpropsがdrillingされているか特定してください
2. `design.md`に以下を記録してください：
   - どのpropsがdrillingされているか
   - どのコンポーネントが「中継しているだけ」の状態か
   - この構造で新しいユーザー情報（例：`email`）を追加する場合、何箇所の変更が必要か

※この演習ではContextを使った解決は行いません。問題を「体感」することが目的です。

---

## ヒント

- 中間コンポーネント（Layout, Header）のpropsを見てみましょう
- 「このpropsは本当にこのコンポーネントで使われているか？」という視点で見てください
