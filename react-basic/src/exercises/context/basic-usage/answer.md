# 回答

## 質問1: Contextの「3点セット」の役割

### createContext

コンテキストを生成する。

### Provider

コンテキストのスコープを定義し、ラップしたコンポーネント内でコンテキストで管理した情報にアクセス可能とさせる。

### useContext

コンテキストにアクセスする。

---

## 質問2: Contextとグローバル変数の違い

ContextはスコープがProvider配下に限定されるのに対して、グローバル変数はどこからでもアクセス可能。

---

## 質問3: useContextがnullを返すケース

Provider外の箇所で、コンテキストにアクセスされた時。

---

## 評価コメント（2026-01-27）

### 質問1
✅ 正しい理解です。各要素の役割を適切に捉えています。

### 質問2
✅ スコープの限定という核心を理解しています。補足：具体例として、「管理者用ダッシュボード」と「一般ユーザー用画面」で異なるUserProviderを使い分けることで、同じUserContextでも異なるユーザー情報を提供できます。グローバル変数ではこのような「同じ名前でスコープごとに異なる値」を実現できません。

### 質問3
✅ 正解です。Provider外でアクセスされた場合にcreateContextで指定したデフォルト値（この場合`null`）が返されます。
