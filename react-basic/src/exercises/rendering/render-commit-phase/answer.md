# 言語化演習の回答

## 1. Render phaseとCommit phaseの役割の違い

<!-- ここに回答を記述してください -->
* Render phaseはこれから表現したいDOMの設計図（コンポーネント）を生成し、既存DOMと差分を検証するフェーズであるのに対して、Commit phaseはRender phaseによって判定された差分を実際のDOMに反映させるフェーズ。



## 2. 「再レンダリング」とは何か、DOM更新との関係

<!-- ここに回答を記述してください -->
* コンポーネントの状態が変わることによって再度コンポーネントを計算すること。必ずしもDOMが変更するわけではなく、あくまで計算結果が実DOMと差分がある場所のみ変更が入る。

### 「状態が変わること」以外に、再レンダリングが発生するトリガーは他にありますか？answer.mdに追記してください。  
* 上記の状態とはステートのことを指していました。それ以外だと、以下が挙げられます。
  * 要素のタグに変更が入った時（div→sectionなど）
  * 要素の属性が変更された時
  * Componentのpropsが変更された時


## 3. useLayoutEffectとuseEffectの実行タイミング

<!-- ここに回答を記述してください -->
* useLayoutEffectは差分がコミットされた直後に実行され、useEffectは差分がコミットされブラウザの描画k完了直後に実行される。
