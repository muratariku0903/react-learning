# Reconciliation（差分計算）

## 目的
Reactの差分計算（Reconciliation）の仕組みと、効率的な更新がどのように行われるかを理解する。

## 要件

### 言語化演習（answer.mdに回答）

以下の質問に答えてください。

1. Reconciliationとは何ですか？なぜReactはこの処理を行うのですか？
2. 「仮想DOM」という言葉の本質は何ですか？「仮想DOMだから速い」という説明の問題点を指摘してください。
3. Reactの差分計算における2つの重要なルールを説明してください。（ヒント: 要素のtype、リストのkey）

### 実装演習（design.mdに設計を記載してから実装）

以下の仕様を満たすコンポーネントを実装してください。

**仕様:**
- ボタンで要素のtypeを切り替えられるUIを作成する
  - 例: `<div>` と `<span>` を切り替える
  - 例: `<input type="text">` と `<textarea>` を切り替える
- 切り替え時に、要素内部の状態（入力値など）がどうなるかを観察する
- 観察結果をnotes.mdに記録する

**確認ポイント:**
- 同じtypeの要素間での切り替え（属性のみの変更）
- 異なるtypeの要素間での切り替え（サブツリーの再作成）

## 制約条件
- 状態の変化を観察するため、制御されていない入力要素（uncontrolled component）を使用すること

## ヒント
- React DevToolsで要素の再マウントを確認できます
- 入力欄に文字を入力してから要素を切り替えると、状態の維持/破棄が分かりやすいです
