# keyと同一性

## 目的
リストレンダリングにおけるkeyの重要性と、keyが差分計算（Reconciliation）にどう影響するかを理解する。

## 要件

### 言語化演習（answer.mdに回答）

以下の質問に答えてください。

1. リストをレンダリングする際にkeyが必要な理由を、Reactの差分計算の観点から説明してください。
2. keyにindexを使うことの問題点を具体例とともに説明してください。
3. 「keyが不安定」とはどういう状態ですか？どのような問題が発生しますか？

### 実装演習（design.mdに設計を記載してから実装）

以下の仕様を満たすコンポーネントを実装してください。

**仕様:**
- TODOリストを作成する（項目の追加・削除機能付き）
- 2つのバージョンを実装する:
  1. keyにindexを使用するバージョン
  2. keyに一意のID（uuid等）を使用するバージョン
- 各TODOアイテムには入力欄を含める（タスク名を編集できる）
- リストの先頭に新しいアイテムを追加できるようにする
- 先頭への追加時の挙動の違いを観察し、notes.mdに記録する

**観察ポイント:**
- 先頭にアイテムを追加したとき、入力欄の値はどうなるか？
- コンソールにkeyに関する警告は出るか？
- React DevToolsでコンポーネントの再マウント状況を確認する

## 制約条件
- 入力欄はuncontrolled componentとして実装すること（defaultValue使用）
- 2つのバージョンを切り替えられるUIにすること

## ヒント
- `crypto.randomUUID()` や `Date.now()` で一意のIDを生成できます
- リストの「先頭」に追加するケースで問題が顕著になります
