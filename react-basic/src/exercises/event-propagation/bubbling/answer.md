# 言語化演習 回答

## Q1. バブリングとは何か

<!--
「バブリング」という言葉を使わずに、イベントが要素間でどのように伝わるかを説明してください。
また、なぜ「バブリング」という名前がついたと思いますか？
-->

**回答：**
子から親要素にかけてイベントが伝わる。例えば、親要素とこ要素にクリックイベントが設定してある場合、小要素をクリックすると小要素のクリックイベントが発火したのち、親要素のクリックイベントも発火する。  
上記の例のように、子から親に向かって上にイベントが伝播していく様子がまるで泡が上に上がっていく様子に似ているから「バブリング」と言う名前がついたと思われる。

## Q2. stopPropagationを使う判断基準

<!--
stopPropagation() を使うかどうか迷ったとき、どのような観点で判断すべきでしょうか？
「使うべきケース」と「使わない方がよいケース」をそれぞれ考えてみてください。
-->

**回答：**

### 使うべきケース
親と子にそれぞれ別々のイベントが用意されている場合。  
このイベントが発火した際に意図せず、親のイベントも発火してしまうといった事態を避けるため。
例えば、何かしらのAPIを呼び出すボタンが設置されたダイアログを例に挙げると、ダイアログの親要素にはクリックするとダイアログが閉じるイベントが設定されているケースで、ダイアログの内のボタンを押下すると、意図せず、ダイアログが閉じるといった自体が発生する。


### 使わない方がよいケース
リストなど複数の要素に対するイベント処理を親で一括で管理・制御したいときは親にイベントを伝播させたほうが良い。じゃないと、リストそれぞれに対して同じイベントを適応することになってしまう。
