# デフォルト動作の阻止（preventDefault）

## 目的

ブラウザのデフォルト動作を理解し、`preventDefault()`を使って適切に制御する方法を学ぶ。

---

## 言語化演習

以下の質問に `answer.md` で回答してください。

### Q1. デフォルト動作の例を挙げる

以下の要素それぞれについて、ブラウザがデフォルトでどのような動作をするか説明してください。

- `<a href="...">` をクリックしたとき
- `<form>` の送信ボタンを押したとき
- `<input type="checkbox">` をクリックしたとき

### Q2. SPAにおけるpreventDefaultの重要性

Reactなどを使ったSPA（シングルページアプリケーション）では、なぜフォーム送信時に `preventDefault()` がほぼ必須なのでしょうか？
`preventDefault()` を呼ばなかった場合、アプリケーションにどのような影響があるか考えてください。

---

## 実装演習

`App.tsx` には、ログインフォームとリンク集があります。以下の要件を満たすように修正してください。

### 現状の問題

1. フォームを送信するとページがリロードされてしまう
2. 「利用規約」リンクをクリックすると外部ページに遷移してしまう（本来はモーダルで表示したい）

### 要件

1. フォーム送信時にページがリロードされないようにする
2. 送信処理として `console.log` でユーザー名とパスワードを表示する
3. 「利用規約」リンクをクリックしたとき、ページ遷移せずに `showTermsModal` を `true` にする
4. 設計意図を `design.md` に記述すること

### ヒント

- イベントオブジェクトには `preventDefault()` メソッドがあります
- どのイベント（`onSubmit`? `onClick`?）で何を止めるべきか考えましょう
