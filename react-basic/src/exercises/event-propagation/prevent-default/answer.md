# 言語化演習 回答

## Q1. デフォルト動作の例を挙げる

<!--
以下の要素それぞれについて、ブラウザがデフォルトでどのような動作をするか説明してください。
-->

**回答：**

### `<a href="...">` をクリックしたとき
ページ遷移


### `<form>` の送信ボタンを押したとき
リクエスト送信と画面リロード


### `<input type="checkbox">` をクリックしたとき
クリックした要素にフォーカスが移り、チェック状態が自動で切り替わる

## Q2. SPAにおけるpreventDefaultの重要性

<!--
Reactなどを使ったSPA（シングルページアプリケーション）では、なぜフォーム送信時に preventDefault() がほぼ必須なのでしょうか？
preventDefault() を呼ばなかった場合、アプリケーションにどのような影響があるか考えてください。
-->

**回答：**
ブラウザのデフォルトの挙動によってページが再リロードされてしまうと、stateの状態がリセットされてしまうので、意図しないタイミングでの表示コンテンツの初期化が発生してしまう。
こうした事態を抑えるために、ブラウザのデフォルトの挙動はできるだけ制限しておくためpreventDefault()がほぼ必須となっている。

---

### 評価コメント

**Q1:** 概ね正しい。`<a>`のページ遷移、`<form>`のリクエスト送信とリロードは正確。`<input type="checkbox">`については、チェック状態の切り替えが主なデフォルト動作であり、その点は触れられている。フォーカスの移動はデフォルト動作というよりブラウザの一般的な挙動なので、厳密には区別できるとなおよい。

**Q2:** 正しい。SPAではページ全体をリロードせずにJavaScriptでUIを制御しているため、フォーム送信のデフォルト動作（ページリロード）が発生するとstateがすべてリセットされてしまう、という核心を正しく捉えている。
