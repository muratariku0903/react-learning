# 設計メモ

## 実装方針

<!-- どのイベントで、何のデフォルト動作を止めるか記述してください -->
* formにセットしてるハンドラにてページリロードのデフォルト動作を制限
* 「利用規約」リンクのイベントにてページ遷移のデフォルト動作を制限

---

### 評価コメント

設計意図と実装が一致している。`onSubmit` で `e.preventDefault()` を呼びページリロードを防止し、`<a>` の `onClick` で `e.preventDefault()` を呼びページ遷移を防止している。それぞれ適切なイベントで適切なデフォルト動作を止めており、問題なし。state配置・再レンダリングについても適切。
