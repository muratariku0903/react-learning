# 言語化演習の回答

## 1. startTransition でラップした更新の特性

<!--
ラップした更新とラップしない更新の違いを説明してください。
-->
startTransition でラップした更新は「緊急度が低い更新」として扱われ、状況によって後回し・中断・置き換えされやすい（例: 検索結果の再描画）。その間も入力などの緊急度が高い更新は優先されやすい。ラップしない更新は緊急度が高い更新として優先的に反映されやすい。

✅ 正しい理解です。「中断・置き換え」という点まで言及できているのが良いです。連続して入力すると前のtransitionはキャンセルされ、最新のものだけが反映されます。



## 2. isPending の役割とUX改善

<!--
isPendingが何を表すのか、どのようなUX改善ができるのかを説明してください。
-->
isPendingはstartTransitionでラップした更新がまだ完了しておらず途中であることを表す。
isPendingを使うことで、更新が途中であるかの判別がつくので、startTransitionで後回しにした処理の更新状態に応じてUIを切り替えることが可能。

✅ 正しい理解です。実装で行ったように、`opacity`を下げて「処理中」を視覚的に伝えるのは良いUXパターンです。


## 3. Suspense と useTransition の組み合わせ

<!--
ローディング表示の挙動がどう変わるか、Suspense単体の場合と比較して説明してください。
-->
Suspense単体だと、更新によって子がsuspendするとSuspense境界はfallback表示に切り替わりやすく、検索結果がローディング表示に置き換わることがある。
一方、useTransitionを使うとその更新は非緊急として扱われるため、更新中は前の検索結果を表示したままにしつつ、準備ができたら新しい結果へ切り替える、といった挙動にしやすい（必要ならisPendingで「更新中」だけを出せる）。

✅ 正しい理解です。「前の検索結果を表示したまま」という点が重要です。これにより画面がガクッと切り替わる不快感を避けられます。
