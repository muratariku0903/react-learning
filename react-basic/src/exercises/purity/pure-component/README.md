# 純粋なコンポーネントとは

## 目的

この演習では、Reactにおける「純粋なコンポーネント」の概念を理解します。
純粋なコンポーネントとは何か、なぜReactがこれを要求するのかを学びます。

## 要件

### 言語化演習

以下の質問に `answer.md` で回答してください。

1. **純粋関数の定義**
   - 数学やプログラミングにおける「純粋関数」とは何ですか？
   - 純粋関数が満たすべき2つの条件を説明してください。

2. **Reactにおける純粋性**
   - Reactコンポーネントが「純粋である」とはどういう意味ですか？
   - `(props, state) → JSX` という式が意味することを自分の言葉で説明してください。

3. **純粋性が必要な理由**
   - Reactが純粋性を要求する理由を3つ挙げ、それぞれ具体的に説明してください。
   - 特に「再レンダリングが何度起きても安全」とはどういうことか、具体例を交えて説明してください。

### 実装演習

`App.tsx` には3つのコンポーネントが実装されています。
それぞれが純粋かどうかを判定し、非純粋なものは純粋に書き換えてください。

設計方針は `design.md` に記載してから実装してください。

1. 各コンポーネントが純粋か非純粋かを判定する
2. 非純粋なコンポーネントを特定し、なぜ非純粋なのかを説明する
3. 非純粋なコンポーネントを純粋に書き換える

## 制約条件

- 外部ライブラリの使用は禁止
- コンポーネントの見た目（出力されるJSX構造）は変更しない
- 機能要件を維持したまま純粋性を確保する

## ヒント

- 純粋関数は「同じ入力に対して常に同じ出力を返す」
- 純粋関数は「外部の状態を変更しない（副作用がない）」
- レンダリング中にやってはいけないことは何か考えてみましょう
