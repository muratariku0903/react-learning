# 条件分岐フィールドと再レンダリング最適化（アンチパターン修正）

## 目的
`watch` を多用すると、フォーム全体がフィールドを1文字打つたびに再レンダリングされてしまう問題に気付き、
`useWatch` / `Controller` / 子コンポーネントへの分割で **必要なところだけ再描画する** 設計を体感する。
さらに、**ある選択肢を選んだ時だけ表示される追加項目** をどう扱うか（フィールドの登録解除との兼ね合い）を学ぶ。

## 題材
保険申し込みフォーム:
- `plan`: `"basic" | "standard" | "premium"`
- `hasFamily`: boolean、true なら `family[]` の入力欄が出る
- `plan === "premium"` の時だけ `addOns`（チェックボックス群）が出る
- 画面右側にリアルタイムの「見積もり金額」を表示する

`App.tsx` には **わざとアンチパターン** で書かれた実装があります。これを修正するのが演習です。

## 言語化演習

以下の質問に `answer.md` で回答してください。

### Q1.
親コンポーネントで `const values = watch();` と書いた時に発生する再レンダリングの挙動を、
`useWatch` を子コンポーネントで使った場合と比較して説明してください。
**「どのコンポーネントが」「どのタイミングで」再レンダリングされるか** を主語にして書くこと。

### Q2.
条件付きで表示されるフィールド（例：`hasFamily` が true の時だけ `family[]` を表示）の場合、
`shouldUnregister` を `true` にするか `false` にするかで、
**「条件を一度 true にして値を入力 → false に戻して再度 true にする」** という操作の結果がどう変わるかを述べてください。
両方の挙動の **使い分け基準** をあなたなりに提案してください。

### Q3.
`useWatch({ name: "plan" })` と `watch("plan")` は機能的には似ていますが、
RHF 公式が **子コンポーネント内では `useWatch` を推奨** しているのはなぜか、
親→子への props 経由 vs 子で直接購読、の観点で説明してください。

## 実装演習

### 要件
`App.tsx` には次のアンチパターンが入っています:
1. 親で `const values = watch();` を呼んでおり、何を打っても全フォームが再レンダリングされる
2. 「見積もり金額」表示も親の中に直接書かれており、入力1文字ごとに再計算される
3. `family[]` の表示／非表示で `shouldUnregister` を考えていないため、戻った時の挙動が曖昧
4. `addOns` の表示制御に `if (watch("plan") === "premium")` を直接書いている

これを次のように修正してください:

- `EstimatePanel` を子コンポーネントとして切り出し、その中で `useWatch` を使って必要なフィールドだけ購読する
- `FamilySection` を子コンポーネントに切り出し、`useFieldArray` で `family[]` を扱う
- `AddOnsSection` を子コンポーネントに切り出し、`plan` を `useWatch` で購読して表示制御する
- 親 `App.tsx` には `useForm` と `<form>` 構造、そして子コンポーネントの呼び出しだけが残る状態にする
- `shouldUnregister` の方針を `design.md` に書く（どちらを選んだか / その理由）
- 動作確認のために、**各コンポーネントで `console.count("EstimatePanel render")` のように出力** し、
  どこが再レンダリングされたかをコンソールで観察できるようにする

### 制約
- 親 `App.tsx` で `watch()` を **呼ばない**
- 「見積もり金額」を `useState` で持たない（フォーム値から導出する）
- `Controller` を使う必要がある場面を1箇所以上見つけて使う

### ヒント
- `useWatch` は `name` を配列で渡すこともできる
- 子コンポーネントで `control` を取りたい場合は、props で渡すか `useFormContext` を使う
- `console.count` はコンポーネントの本体に直接書けば毎レンダリングで増える
