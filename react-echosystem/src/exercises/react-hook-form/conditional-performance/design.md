shouldUnregisterはこちらはフォルスを設定して、コンポーネントがアンマウントされた場合でも値をリセットさせないようにしようかなと思っています。家族を追加するチェックボックスを間違えて外しちゃった時に、すべての入力が初期化されてしまうと手間だと思うので、入力は保持しておきたいです。

申し込むボタンを押した時に、今回は多分ミューテーションなどは行わないと思うんですが、その際に家族を追加した場合、家族の入力フォームの相関がちゃんと取れているかをチェックすればいいかなと思っています。

コントロールを渡すのか、それともuseform contextを使うのかについてですが、今回そんなにネスト深くないんで、コントロールをシンプルに渡して、それをusewatchに紐付けるやり方でいいような気がしてます。

なので、各コンポーネントにはコントロールを渡して紐付けるやり方で一旦やってみようと思います。


基本的に入力を必要とするようなパーツっていうのはコントローラーを使うことで高読範囲っていうのを限定することができます。入力を必要としない単純に表示するだけのものに関してはウォッチで対応しようかなと思っています。
