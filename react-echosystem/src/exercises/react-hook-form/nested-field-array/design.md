useFormContext の使いどころ
親のアイテム図はApp.tsxのuseFieldArrayで呼び出して、言い呼び出しとか管理して、オプション図はItemRowのコンポーネント側で同じくuseFieldArrayで管理します。useFormContextを使って、ItemRowのコンポーネントから直接コントローラーを参照するようにします。
こうすることで、Options の追加や削除が Items に影響を与えなくなるため、レンダリングの範囲を限定することができます。

再レンダリング抑制のための工夫
 useFormContext を使って直接 control をコンポーネントで取得します。こうすることによって、再レンダリングする範囲をコンポーネントに留めることができます。

動的 name の組み立て
ユーズウォッチは使うかどうかわかんないです。使わなくてもいけるんじゃないって正直思ってます。React-Memoを使うとしたら、ItemRow ですかね。親のリストの構造に変更が生じた場合に、個コンポーネントも再レンダリングされるので、\n\nその際にキャッシュした値を返せればパフォーマンス向上すると思うので、使うんだったら ItemRow かなと思います。
