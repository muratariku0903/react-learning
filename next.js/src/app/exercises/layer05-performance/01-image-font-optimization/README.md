# 演習5-1: 画像・フォント最適化

## 目的

Next.js が提供する `next/image` と `next/font` の最適化機能を理解し、**なぜ標準の HTML 要素（`<img>`, `<link>`）ではなくこれらを使うべきなのか**を、パフォーマンスの観点から体感する。

---

## 言語化演習

以下の問いに対して、`answer.md` に自分の言葉で回答してください。

### Q1: `next/image` の最適化メカニズム

`next/image` コンポーネントは、通常の `<img>` タグと比べて**具体的にどのような最適化**を行っているか？以下の観点を踏まえて列挙してください。

- 画像フォーマットの変換
- サイズの自動調整
- 遅延読み込み（Lazy Loading）
- レイアウトシフトの防止
- キャッシュ

### Q2: `sizes` プロパティの役割

`next/image` の `sizes` プロパティは何のために存在するか？`sizes` を指定しない場合にどのような問題が起きうるか、ブラウザの画像選択の仕組みと関連づけて説明してください。

### Q3: `next/font` がもたらすメリット

`next/font` を使わずに Google Fonts を `<link>` タグで読み込んだ場合と比較して、`next/font` はどのような問題を解決するか？**パフォーマンス**と**プライバシー**の両面から説明してください。

### Q4: CLS（Cumulative Layout Shift）との関係

画像とフォントの読み込みは CLS（Cumulative Layout Shift）にどう影響するか？`next/image` と `next/font` はそれぞれどのようにして CLS を防いでいるか説明してください。

---

## 実装演習

### 実装1: `<img>` vs `next/image` の比較ページ

同じ画像を以下の2パターンで表示するページを作成してください。

1. **通常の `<img>` タグ**で表示
2. **`next/image`** で表示

#### 要件
- 外部の画像URL（例: Unsplash）または `/public` に配置したローカル画像を使用
- それぞれの表示方法の下に、**何が違うのか**を説明するテキストを添える
- DevTools の Network タブで、画像リクエストの違い（フォーマット、サイズ）を確認できるようにする
- `sizes` プロパティを適切に設定し、レスポンシブ対応すること

#### ヒント
- 外部画像を使う場合は `next.config.ts` の `images.remotePatterns` に設定が必要
- `fill` プロパティを使う場合、親要素に `position: relative` が必要

### 実装2: `next/font` によるフォント最適化

Google Fonts から任意のフォントを読み込み、ページに適用してください。

#### 要件
- `next/font/google` からフォントをインポートして使用する
- `display: 'swap'` の設定を行い、その意味を理解する
- フォントの適用範囲を**この演習ページ内のみ**に限定すること（ルートレイアウトを変更しない）
- 複数のフォントウェイト（例: 400, 700）を使い分ける実装をすること

#### ヒント
- `next/font/google` でフォントを初期化する際、`subsets` と `weight` の指定方法を調べてみましょう
- フォントの CSS 変数（`variable`）を使った適用方法も検討してみましょう
